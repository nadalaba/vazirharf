3:T2fb9,<h3>What is the license for using the Vazirharf font?</h3>
<p>Vazirharf, like its original project Vazirmatn, is released under the OFL version 1.1 license. Anyone can use this font for any purpose.</p>
<h3>What is the official page address for Vazirharf?</h3>
<p>Currently, the official page is:<br>
<a href="https://nadalaba.github.io/vazirharf">https://nadalaba.github.io/vazirharf</a></p>
<h3>Why was the name changed from Vazirmatn to Vazirharf?</h3>
<p>The name change to Vazirharf was made to avoid confusion with the original font, which was widely distributed via package managers but is no longer being updated. The new name reflects both continuity and evolution:</p>
<ul>
<li>"Vazir": Preserves the legacy of the original font.</li>
<li>"Harf": Means "letter" or "character", indicating a shift from single-glyph combinations to letter-based compositions.</li>
</ul>
<h3>What is the difference between Vazirharf and Vazirmatn?</h3>
<p>The main difference is that Vazirharf splits the "Lam-Alef" combination into two separate glyphs using contextual substitution and the 'rclt' feature, whereas Vazirmatn uses 'liga' and 'rlig' features for these combinations.</p>
<h3>Who Should Use Vazirharf?</h3>
<p>Use <strong>Vazirharf</strong>:</p>
<ul>
<li>When you need to disable ligatures (because you want feedback for each character you type),but you still want to preserve the original ligature shape of Lam-Alef <code>لا</code>.
This is especially useful on typing platforms that do not rely on a traditional input box, such as <a href="https://monkeytype.com">monkeytype</a>. You can clearly see the difference <a href="https://github.com/monkeytypegame/monkeytype/pull/7552">here</a>.</li>
<li>If you require ongoing updates and support.</li>
</ul>
<h3>Is Vazirmatn the same as Vazir?</h3>
<p>Starting from version 32, the Vazir font was renamed to Vazirmatn.</p>
<h3>What are the differences between Vazirmatn and Vazir?</h3>
<p>In Vazirmatn, font sizes—especially bold styles—have been increased. The weights were rebuilt with a new approach (revised thickness, contrast, and spacing). New weights were added. Many symbols were added, and some previous digits, letters, and symbols were redesigned. The font’s development method changed, scripts were rewritten, folder and file structures were completely revamped to be simpler and more readable. Naming conventions for variant versions changed. Latin glyphs were expanded and European languages (from Roboto font) were added. Many design and technical issues were resolved. Vazirmatn 32 marks a new chapter and a major step forward for this typeface. For a list of changes, see the <a href="https://github.com/nadalaba/vazirharf/blob/master/CHANGELOG.md">CHANGELOG.md</a>.</p>
<h3>What languages does this font support?</h3>
<p>As of version 27 and later, Vazirharf supports Persian, Arabic, Kurdish, Pashto, Urdu, Gilaki, Uzbek, Kazakh, and Balochi.</p>
<h3>What category of fonts does Vazirharf belong to?</h3>
<p>It belongs to the Sans category.</p>

<h3>How do I install the font on Windows?</h3>
<p>Open the main or compressed file downloaded from the font’s homepage. Then go to the fonts/ttf folder, open the files, and click the Install button in the pop-up window. Alternatively, select all ttf files and move them to the windows/fonts folder.</p>
<h3>Why are numbers displayed in Arabic or Latin instead of Persian?</h3>
<p>The standard Vazirharf font displays numbers in Persian if they are typed using the Persian input standard. If you see English or Arabic digits, it means they weren’t typed in Persian.<br>
Examples:<br>
Persian: ۱۲۳۴۵۶۷۸۹۰<br>
Arabic: ١٢٣٤٥٦٧٨٩٠<br>
Latin: 1234567890</p>
<p>A standard font should not alter the original text or correct input errors.</p>
<h3>Why are ۱ ۲ ۳ ۷ ۸ ۹ Persian but ٤ ٥ ٦ ٠ are not?</h3>
<p>These digits differ between Arabic and Persian. For example, Arabic zero is usually solid, while Persian zero is hollow. To display Persian digits, they must be typed using the Persian standard code.</p>
<h3>I don’t care if the original text uses Latin or Arabic digits. How can I force Vazirharf to display Persian digits?</h3>
<p>There are two methods:</p>
<ul>
<li>Use fonts in the <code>misc/Farsi-Digits</code> folder, where all English and Arabic digits are rewritten as Persian. This isn’t standard but solves the issue.</li>
<li>Use a Stylistic Set: If your software supports OpenType features, enable ss01 Farsi Digits in Stylistic Sets, or use this CSS code on the web:</li>
</ul>
<pre><code class="language-css">.sample_farsi_digits {
    font-feature-settings: "ss01";
}
</code></pre>
<h3>Why aren’t the digits monospaced in Vazirharf?</h3>
<p>Monospaced digits mean, for example, that the width of the digit 1 is equal to the width of the digit 5. In this case, the width of the two numbers below will be the same:<br>
۱۱۱۱۱<br>
۵۵۵۵۵<br>
This format is more suitable for reports. To display monospaced digits, if the software you’re using supports OpenType feature settings, select the Tabular Numeric option, or use the following CSS code in a web environment:</p>
<pre><code class="language-css">.sample_tabular_numbers {
    font-feature-settings: "tnum";
}
</code></pre>
<h3>How can I use monospaced digits or fully Persian digits in LibreOffice?</h3>
<p>In LibreOffice, go to the Format menu, select Character, then in the Font tab, click the Features button. Choose Stylistic Set 1 and Tabular Number for this purpose.</p>
<h3>How can I use monospaced digits or fully Persian digits in Inkscape?</h3>
<p>From the Text menu, select Text and Font..., then in the Features tab, under Numeric, choose Tabular, or under Feature Setting, select ss01.</p>
<h3>How can I display fully Persian and monospaced digits simultaneously on a webpage using CSS?</h3>
<pre><code class="language-css">.sample_ss01_tnum {
    font-feature-settings: "ss01", "tnum";
}
</code></pre>
<h3>Which Latin font is combined with Vazirharf?</h3>
<p>It’s combined with Roboto from Google. Roboto is a free font released under the Apache License version 2. <a href="https://github.com/googlefonts/roboto">https://github.com/googlefonts/roboto</a>.</p>
<h3>Where is the version of this font without Latin letters and symbols?</h3>
<p>It’s located in the <code>misc/Non-Latin</code> folder.</p>
<h3>How can I merge Vazirharf with my preferred font?</h3>
<p>You’ll need a font editing software. For example, you can install FontForge, open the Vazirharf file, then from the <code>Element</code> menu, choose <code>Merge Fonts...</code> and combine it with your desired font.</p>
<h3>Why do the top and bottom of text appear misaligned compared to other Latin fonts in the app I’m developing, and how can I fix it?</h3>
<p>This issue is common with Persian/Arabic fonts. It’s because many Persian/Arabic letters like ("ح", "ر", "ی", etc), extend below the baseline, and diacritics are added above and below letters. As a result, Persian/Arabic letters often require more vertical space than Latin ones. For those who need a shorter version, an alternative is available in the UI folder of the package, which has reduced height.</p>
<h3>What is the UI version?</h3>
<p>In this version, the font height is reduced to minimize the height difference between Latin and Persian/Arabic text. It’s suitable for those who need such formatting in an application's user interface.</p>
