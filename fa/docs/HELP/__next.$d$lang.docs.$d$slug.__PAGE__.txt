3:T459a,<h3>مجوز استفاده از فونت وزیرحرف چیست؟</h3>
<p>فونت وزیرحرف همچون پروژه اصلی خود، وزیرمتن، تحت مجوز OFL نسخه ۱.۱ منتشر شده است. هر کسی می‌تواند از این فونت برای هر نوع مقصودی استفاده کند.</p>
<h3>آدرس صفحه رسمی فونت وزیرحرف چیست؟</h3>
<p>در حال حاضر این آدرس است که می‌توان شکل قلم را نیز در آن مشاهده کرد:<br>
<a href="https://nadalaba.github.io/vazirharf/">https://nadalaba.github.io/vazirharf</a></p>
<h3>چرا نام وزیرمتن به وزیرحرف تغییر پیدا کرده است؟</h3>
<p>تغییر نام به وزیرحرف برای جلوگیری از سردرگمی با فونت اصلی انجام شده است؛ فونتی که به‌طور گسترده از طریق مدیرهای بسته توزیع شده اما دیگر بروزرسانی نخواهد شد.
نام وزیرحرف هم‌زمان نشان‌دهندهٔ تداوم و تحول است:</p>
<ul>
<li>«وزیر»: حفظ‌کنندهٔ میراث فونت اصلی.</li>
<li>«حرف»: به‌معنای «حرف» یا «کاراکتر»، که نشان‌دهندهٔ تغییر رویکرد از ترکیب‌های تک‌گلیف به ترکیب‌های مبتنی بر حروف است.</li>
</ul>
<h3>تفاوت وزیرحرف با وزیرمتن چیست؟</h3>
<p>تفاوت اصلی در این است که وزیرحرف ترکیب‌های لام-الف را با استفاده از جایگزینی زمینه‌ای و ویژگی 'rclt' به دو گلیف جداگانه تقسیم می‌کند، در حالی که در وزیرمتن از ویژگی‌های 'liga' و 'rlig' برای این ترکیب‌ها استفاده شده است.</p>
<h3>چه کسانی باید از وزیرحرف استفاده کنند؟</h3>
<p>از <strong>وزیرحرف</strong> استفاده کنید:</p>
<ul>
<li>زمانی که لازم دارید لیگچرها را غیرفعال کنید (چون می‌خواهید برای هر کاراکترِ تایپ‌شده بازخورد جداگانه داشته باشید)، اما همچنان می‌خواهید شکل اصلی لیگچر لام‌الف «لا» حفظ شود.
این موضوع به‌ویژه در پلتفرم‌های تایپ که از جعبه ورودی سنتی استفاده نمی‌کنند، مثل <a href="https://monkeytype.com">monkeytype</a>، بسیار کاربردی است. تفاوت را می‌توانید <a href="https://github.com/monkeytypegame/monkeytype/pull/7552">اینجا</a> ببینید.</li>
<li>اگر به پشتیبانی و به‌روزرسانی‌های مداوم نیاز دارید.</li>
</ul>
<h3>آیا وزیرمتن همان وزیر است؟</h3>
<p>از نسخه ۳۲ فونت وزیر به وزیرمتن تغییر نام پیدا کرده است.</p>
<h3>وزیرمتن چه تفاوتی با وزیر دارد؟</h3>
<p>در فونت وزیرمتن اندازه قلم به‌ويژه در حالت‌های ضخیم بزرگ‌تر شده است. وزن‌های آن از نو با رویکردی جدید ساخته شده‌اند (در ضخامت، کنتراست و فاصله‌ها بازنگری شده). تعدادی وزن جدید به آن اضافه شده است. علائم متعددی به آن افزوده و برخی ارقام، حروف و علائم قبلی نیز بازطراحی شده‌اند. شیوه تولید و توسعه فونت دچار تغییر گشته و اسکریپت‌ها بازنویسی شده‌اند. ساختار پوشه‌بندی و فایل‌ها کاملا تغییر کرده و البته ساده‌تر و خواناتر شده است. نحوهٔ نامگذاری نسخه‌های متفرقه تغییر کرده است. گلیف‌های لاتین کامل‌تر و زبان‌های اروپایی (از فونت روبوتو) نیز به آن اضافه شده است. بسیاری از اشکالات طراحی و فنی آن برطرف شده است. در هر صورت وزیرمتن ۳۲ فصلی جدید و یک گام بزرگ برای این تایپ‌فیس محسوب می‌شود. برای اطلاع از فهرست تغییرات، فایل تاریخچه را مشاهده نمایید. <a href="https://github.com/nadalaba/vazirharf/blob/master/CHANGELOG.md">«CHANGELOG.md»</a></p>
<h3>این فونت از چه زبان‌هایی پشتیبانی می‌کند؟</h3>

<h3>چرا ارقام/اعداد را به صورت عربی یا لاتین نمایش می‌دهد و فارسی نمایش نمی‌دهد؟</h3>
<p>فونت وزیرحرف استاندارد در صورتی که ارقام را طبق استاندارد ورودی فارسی نوشته باشید به همان صورت فارسی نمایش می‌دهد. اگر شما انگلیسی یا عربی می‌بینید به این معناست که آن‌ها در هنگام نوشتن فارسی نبوده‌اند.<br>
فارسی: ۱۲۳۴۵۶۷۸۹۰<br>
عربی: ١٢٣٤٥٦٧٨٩٠<br>
لاتین: 1234567890</p>
<p>یک فونت استاندارد نباید چیزی غیر از اصل متن را نمایش دهد و همچنین نباید اشتباهاتِ هنگام ورود و نگارش متن را اصلاح نماید.</p>
<h3>چرا ۱ ۲ ۳ ۷ ۸ ۹ فارسی است اما ٤ ٥ ٦ ٠ فارسی نیست؟</h3>
<p>شکل این ارقام در عربی با فارسی متفاوت است. مثلا در عربی معمولا صفر توپُر است و در فارسی توخالی. برای فارسی بایستی اعداد با کد استاندارد فارسی نوشته شده باشند.</p>
<h3>برایم مهم نیست که در اصلِ متن، اعداد را به صورت لاتین یا عربی نوشته‌اند. چگونه فونت وزیرحرف را مجبور به نمایش فارسی اعداد کنم؟</h3>
<p>در حال حاضر دو راه وجود دارد:</p>
<ul>
<li>استفاده از فونت‌های موجود در پوشه <code>misc/Farsi-Digits</code> که تمامی ارقام انگلیسی و عربی آن با فارسی بازنویسی شده‌اند. اگر چه این روش استاندارد نیست اما می‌تواند مشکل شما را حل کند.</li>
<li>استفاده از Stylistic Set: کافیست اگر در برنامه مورد استفاده قابلیت تنظیم Open Type Features پشتیبانی می‌شود گزینه ss01 Farsi Digits در Stylistic Sets را انتخاب کنید و یا آنکه در محیط وب از این کد css استفاده کنید:</li>
</ul>
<pre><code class="language-css">.sample_farsi_digits {
    font-feature-settings: "ss01";
}
</code></pre>
<h3>چرا در فونت وزیرحرف ارقام هم‌عرض نیستند؟</h3>
<p>ارقام هم‌عرض یعنی برای مثال اندازه عرض ۱ با ۵ برابر باشد. در اینصورت عرض دو عدد زیر یکسان خواهد بود:<br>
۱۱۱۱۱<br>
۵۵۵۵۵<br>
این حالت بیشتر برای گزارشات مناسب است. برای نمایش ارقام هم‌عرض کافیست اگر در برنامه مورد استفاده قابلیت تنظیم Open Type Features پشتیبانی می‌شود گزینه Tabular Numeric را انتخاب کنید و یا آنکه در محیط وب از این کد css استفاده کنید:</p>
<pre><code class="language-css">.sample_tabular_numbers {
    font-feature-settings: "tnum";
}
</code></pre>
<h3>چگونه در لیبره‌آفیس LibreOffice از امکان ارقام هم‌عرض و یا تمام ارقام فارسی استفاده کنم؟</h3>
<p>در لیبره‌آفیس از منوی Format گزینه Character را انتخاب و سپس در برگه Font دکمه Features را انتخاب نمایید. دو گزینه Stylistic Set 1 و Tabular Number برای این منظور می‌باشند.</p>
<h3>چگونه در اینک‌اسکیپ Inkscape از امکان ارقام هم‌عرض و یا تمام ارقام فارسی استفاده کنم؟</h3>
<p>از منوی Text گزینه Text and Font... رو انتخاب و سپس در برگه Features قسمت Numeric گزینه Tabular و یا در قسمت Feature Setting گزینه ss01 را انتخاب نمایید.</p>
<h3>چگونه در صفحهٔ وب با استفاده از css همزمان ارقام کاملا فارسی و هم‌عرض (با عرض یکسان) داشته باشم؟</h3>
<pre><code class="language-css">.sample_ss01_tnum {
    font-feature-settings: "ss01", "tnum";
}
</code></pre>

<p>در پوشه <code>misc/Non-Latin</code> قرار دارند.</p>
<h3>چگونه فونت وزیرحرف را با فونت دلخواهم ترکیب کنم؟</h3>
<p>برای این کار شما باید از یک نرم‌افزار ویرایشگر فونت کمک بگیرید. برای مثال می‌توانید فونت‌فورج FontForge را نصب کرده و با آن فایل وزیرحرف را باز نموده، سپس از منوی <code>Element</code> گزینه <code>Merge Fonts...</code> را برگزیده و با فونت دلخواه خودتان ادغام نمایید.</p>
<h3>چرا در برنامه‌ای که توسعه می‌دهم بالا و پایین نوشته‌ها در مقایسه با دیگر فونت‌های لاتین تراز به نظر نمی‌رسد و باید آن را تنظیم کنم؟</h3>
<p>این مشکل معمولا در مورد فونت‌های فارسی/عربی وجود دارد. دلیلش اینست که بخش عمده‌ای از حروف فارسی مثل «ح ر ی ...» در زیر خط کرسی یا زمینه قرار دارند و همچنین به بالا و پایین حروف، اِعراب را هم باید اضافه کرد. در نتیجه حروف فارسی غالبا به فضای بیشتری نسبت به حروف لاتین برای نمایش احتیاج دارند. اما برای آن دسته از کسانی که نیاز به نسخه کوتاه دارند نسخهٔ دیگری در پوشه UI بسته قرار گرفته است که دارای ارتفاع کمتری می‌باشد.</p>
<h3>نسخه UI چیست؟</h3>
<p>در این نسخه از ارتفاع فونت کاسته شده است تا اختلاف ارتفاع بین لاتین و فارسی کم شود. این نسخه برای افرادی مناسب است که نیاز به چنین حالتی در رابط کاربری User Interface برنامه‌ها دارند.</p>
<h3>چگونه می‌توانم در میزکارهای مبتنی بر Fontconfig (بیشتر سیستم عامل‌های Unix-like)، فونت پیش‌فرض را وزیرحرف قرار دهم؟</h3>
